# 3. การเชื่อมต่อฐานข้อมูล

HerbReady ต้องเชื่อมต่อกับฐานข้อมูลของระบบ HIS (Hospital Information System) ของโรงพยาบาลเพื่อดึงข้อมูลผู้ป่วยและการจ่ายยา

---

## ฐานข้อมูลที่รองรับ

- **MySQL 8.0+** — รองรับเท่านั้นในเวอร์ชันปัจจุบัน
- ระบบ HIS ที่รองรับ: HosXP, PMQA, หรือระบบอื่นที่ใช้ MySQL

> **หมายเหตุ:** ฐานข้อมูลต้องอนุญาตให้เชื่อมต่อจากเครื่องที่รัน HerbReady (ตรวจสอบ firewall และสิทธิ์ผู้ใช้)

---

## การตั้งค่าการเชื่อมต่อฐานข้อมูล

เมื่อเปิดโปรแกรม HerbReady เป็นครั้งแรก หรือเมื่อยังไม่ได้ตั้งค่าฐานข้อมูล ให้ทำดังนี้:

### ขั้นตอนที่ 1: เปิดการตั้งค่า

1. คลิกที่ **ไอคอนรูปเฟือง** (⚙️) มุมขวาบนของโปรแกรม
2. เลือกเมนู **"การตั้งค่า"** หรือ **"Settings"**
3. จะเห็นแท็บสำหรับการตั้งค่าต่างๆ

### ขั้นตอนที่ 2: กรอกข้อมูลการเชื่อมต่อ

กรอกข้อมูลดังนี้:

| ช่อง | คำอธิบาย | ตัวอย่าง |
|------|----------|----------|
| **Host** | ที่อยู่เซิร์ฟเวอร์ฐานข้อมูล | `127.0.0.1` หรือ `localhost` |
| **Port** | พอร์ต MySQL (ปกติ 3306) | `3306` |
| **Database Name** | ชื่อฐานข้อมูล HIS | `hosxp`, `pmqa` |
| **Username** | ชื่อผู้ใช้ฐานข้อมูล | `root` |
| **Password** | รหัสผ่านผู้ใช้ฐานข้อมูล | `******` |

### ขั้นตอนที่ 3: ทดสอบการเชื่อมต่อ

1. คลิกปุ่ม **"ทดสอบการเชื่อมต่อ"** หรือ **"Test Connection"**
2. รอสักครู่ โปรแกรมจะแจ้งผล:
   - **สำเร็จ:** แสดงข้อความ "เชื่อมต่อสำเร็จ" พร้อมเวอร์ชัน MySQL
   - **ล้มเหลว:** แสดงข้อความผิดพลาด ให้ตรวจสอบข้อมูลอีกครั้ง

### ขั้นตอนที่ 4: บันทึกการตั้งค่า

เมื่อทดสอบสำเร็จ ให้คลิก **"บันทึก"** เพื่อบันทึกการตั้งค่า

ข้อมูลการเชื่อมต่อจะถูกเข้ารหัส (encrypt) และบันทึกในไฟล์ `config.ini` ในโฟล์เดอร์:
- **Windows:** `%LOCALAPPDATA%\HerbReady\`
- **macOS:** `~/Library/Application Support/HerbReady/`
- **Linux:** `~/.local/share/HerbReady/`

---

## การแก้ปัญหาการเชื่อมต่อ

### ปัญหาที่พบบ่อยและวิธีแก้ไข

#### 1. ไม่สามารถเชื่อมต่อได้ (Connection Failed)

**สาเหตุที่เป็นไปได้:**
- ที่อยู่ Host ผิด
- พอร์ตผิด
- MySQL server ไม่ทำงาน
- Firewall ปิดกั้นการเชื่อมต่อ

**วิธีแก้ไข:**
1. ตรวจสอบว่า MySQL server ทำงานอยู่
   - Windows: ตรวจสอบใน Services
   - macOS/Linux: รัน `sudo systemctl status mysql` หรือ `sudo service mysql status`
2. ตรวจสอบ Host และ Port ให้ถูกต้อง
3. ตรวจสอบ Firewall ว่าอนุญาตการเชื่อมต่อพอร์ต 3306

#### 2. ชื่อผู้ใช้หรือรหัสผ่านไม่ถูกต้อง

**วิธีแก้ไข:**
1. ตรวจสอบชื่อผู้ใช้และรหัสผ่าน
2. ตรวจสอบว่าผู้ใช้มีสิทธิ์เชื่อมต่อจากเครื่องที่รัน HerbReady
   ```sql
   -- ตรวจสอบสิทธิ์ใน MySQL
   SHOW GRANTS FOR 'username'@'hostname';
   ```
3. ลองใช้ MySQL Client (เช่น MySQL Workbench, DBeaver) เพื่อทดสอบการเชื่อมต่อ

#### 3. ไม่พบฐานข้อมูลที่ระบุ

**วิธีแก้ไข:**
1. ตรวจสอบชื่อฐานข้อมูลให้ถูกต้อง (case-sensitive)
2. ตรวจสอบว่าผู้ใช้มีสิทธิ์เข้าถึงฐานข้อมูลนั้น
   ```sql
   -- แสดงรายฐานข้อมูลที่ผู้ใช้เข้าถึงได้
   SHOW DATABASES;
   ```

#### 4. Connection Timeout

**วิธีแก้ไข:**
1. ตรวจสอบ network ระหว่างเครื่องกับ MySQL server
2. ลอง ping ไปยัง MySQL server
   ```bash
   ping <hostname>
   ```
3. ตรวจสอบค่า `max_connections` ใน MySQL

## ข้อมูลตารางที่ใช้

HerbReady ใช้ข้อมูลจากตารางต่อไปนี้ในฐานข้อมูล HIS:

| ตาราง | คำอธิบาย | ข้อมูลที่ใช้ |
|-------|----------|-------------|
| `patient` | ข้อมูลผู้ป่วย | HN, CID, pname, fname, lname, pttype |
| `ovst` | การมาพบแพทย์ | VN, HN, vstdate, cur_dep, pttype |
| `opdscreen` | การตรวจร่างกาย | VN, bw, bps, bpd, pulse |
| `opitemrece` | รายการยาที่จ่าย | VN, HN, icode, qty |
| `drugitems` | ข้อมูลยา | icode, name, units |
| `pttype` | ประเภทสิทธิ์ | pttype, name |
| `kskdepartment` | ห้องตรวจ | depcode, department |
| `lab_head` | ใบสั่งแลป | lab_order_number, hn, order_date |
| `lab_order` | ผลแลป | lab_order_number, lab_items_code, lab_order_result |
| `lab_items` | รายการแลป | lab_items_code, lab_items_name |

> **หมายเหตุ:** ชื่อตารางอาจแตกต่างกันในแต่ละระบบ HIS โปรแกรมรองรับการปรับแต่งในการตั้งค่า

---

## การตรวจสอบการเชื่อมต่อสำเร็จ

เมื่อเชื่อมต่อสำเร็จ ในแถบด้านล่างของโปรแกรมจะแสดง:
- สถานะ: **"เชื่อมต่อแล้ว"** (Connected)
- ชื่อฐานข้อมูล
- เวอร์ชัน MySQL

หากสถานะแสดง **"ไม่เชื่อมต่อ"** (Disconnected) ให้คลิกที่แถบสถานะเพื่อเปิดการตั้งค่าและแก้ไข

---

## ความปลอดภัย

- รหัสผ่านฐานข้อมูลจะถูกเข้ารหัสด้วย AES-256 ก่อนบันทึกลงไฟล์
- ไฟล์การตั้งค่าอยู่ในโฟล์เดอร์ของผู้ใช้เท่านั้น
- ไม่ควรแชร์ไฟล์การตั้งค่ากับผู้อื่น

---

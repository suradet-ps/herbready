# 1. ภาพรวมของโปรแกรม HerbReady

## HerbReady คืออะไร?

**HerbReady** เป็นโปรแกรมจ่ายยาสมุนไพรสำหรับโรงพยาบาลและคลินิก พัฒนาด้วยเทคโนโลยี Tauri (Rust + Vue 3) ช่วยให้เจ้าหน้าที่สามารถ:

- จัดการคิวผู้ป่วยประจำวัน
- ค้นหาผู้ป่วยและตรวจสอบประวัติการรับยาสมุนไพร
- ตรวจสอบความเหมาะสมในการจ่ายยา
- แจ้งเตือนปฏิกิริยาระหว่างยาแผนปัจจุบันกับยาสมุนไพร
- ตรวจสอบผลแลปและแจ้งเตือนค่าผิดปกติ
- จ่ายยาและพิมพ์ใบสั่งยา PDF
- ส่งออกข้อมูลเป็น Excel

## ฟีเจอร์หลัก

### 1. ระบบคิวประจำวัน (Daily Queue Processing)
โหลดรายชื่อผู้ป่วยที่มาพบแพทย์แผนไทยในวันที่เลือก แบ่งกลุ่มยาสมุนไพรที่สามารถจ่ายได้:

- **ยาที่จ่ายได้ (Eligible)** — ผู้ป่วยที่ห่างจากการรับยาครั้งสุดท้ายเกินกว่าระยะเวลาที่กำหนด
- **ไม่เคยจ่าย (Never Dispensed)** — ผู้ป่วยที่ไม่เคยรับยาสมุนไพรชนิดนี้เลย
- **ยังไม่ถึงกำหนด (Not Yet Eligible)** — ผู้ป่วยที่เพิ่งรับยาไป ต้องรออีกกี่วัน

### 2. ระบบค้นหาผู้ป่วย (Patient Search)
ค้นหาผู้ป่วยได้ 3 วิธี:
- **หมายเลข HN** — หมายเลขผู้ป่วยของโรงพยาบาล
- **หมายเลขบัตรประจำตัวประชาชน (CID)**
- **ชื่อ-นามสกุล** — ค้นหาด้วยชื่อผู้ป่วย

### 3. ระบบประวัติการจ่ายยา (Dispensing History)
ดูประวัติการจ่ายยาสมุนไพรย้อนหลังตามช่วงวันที่ สามารถ:
- กรองตาม HN, CID หรือชื่อ
- ดูรายละเอียดยาที่จ่ายในแต่ละครั้ง
- ดูย้อนกลับไป 1 ปี

### 4. ระบบแจ้งเตือนปฏิกิริยาระหว่างยา (Herb-Drug Interaction Alerts)
ตรวจสอบว่าผู้ป่วยมีการใช้ยาแผนปัจจุบันที่อาจมีปฏิกิริยากับยาสมุนไพร เช่น:
- ยาลดความดัน + สมุนไพรบางชนิด
- ยาแก้ปวด + สมุนไพรบางชนิด

### 5. ระบบผลแลป (Lab Result Integration)
ดึงผลแลปล่าสุดของผู้ป่วยและเปรียบเทียบกับค่าขีดจำกัดที่กำหนด แจ้งเตือนเมื่อค่าผิดปกติ เช่น:
- ค่าน้ำตาลในเลือดสูงกว่าปกติ
- ค่าไตผิดปกติ

### 6. ระบบจ่ายยาและพิมพ์ใบสั่งยา (Dispense & Print)
- เลือกยาที่จ่ายสำหรับผู้ป่วย
- กำหนดจำนวน
- พิมพ์ใบสั่งยาเป็น PDF
- พิมพ์พร้อมกันหลายใบ (Batch Print)

### 7. ระบบส่งออกข้อมูล (Excel Export)
ส่งออกรายงานการจ่ายยาประจำวันเป็นไฟล์ Excel (.xlsx) สำหรับนำไปใช้งานต่อ

### 8. รองรับหลายภาษา
รองรับทั้ง **ภาษาไทย** และ **ภาษาอังกฤษ**

## หน้าจอหลักของโปรแกรม

โปรแกรมประกอบด้วย **3 แท็บหลัก**:

| แท็บ | ชื่อ | หน้าที่ |
|------|------|--------|
| 1 | **คิววันนี้** | โหลดและประมวลผลคิวผู้ป่วยประจำวัน |
| 2 | **ค้นหาผู้ป่วย** | ค้นหาผู้ป่วยรายบุคคลด้วย HN/CID/ชื่อ |
| 3 | **ประวัติการจ่ายยา** | ดูประวัติการจ่ายยาสมุนไพรย้อนหลัง |

และมี **แผงเลือกยา (Drug Panel)** ที่ด้านขวาของหน้าจอสำหรับเลือกยาที่จ่าย

3. **ตั้งค่าโปรแกรม** — กำหนดค่ายา ห้องตรวจ และกฎต่างๆ ดูที่ [การตั้งค่าโปรแกรม](./4-configuration.md)

## โครงสร้างฐานข้อมูล

HerbReady เชื่อมต่อกับฐานข้อมูล HIS ของโรงพยาบาล ใช้ตารางหลักดังนี้:

| ตาราง | คำอธิบาย |
|-------|----------|
| `patient` | ข้อมูลผู้ป่วย (HN, CID, ชื่อ, นามสกุล) |
| `ovst` | การมาพบแพทย์ (VN, HN, วันที่, ห้องตรวจ) |
| `opdscreen` | การตรวจร่างกาย (น้ำหนัก, ความดัน, ชีพจร) |
| `opitemrece` | รายการยาที่จ่าย (VN, icode, qty) |
| `drugitems` | ข้อมูลยา (รหัสยา, ชื่อยา, หน่วย) |
| `pttype` | ประเภทสิทธิ์การรักษา |
| `kskdepartment` | ข้อมูลห้องตรวจ |
| `lab_head` | ใบสั่งแลป (หมายเลขใบ, HN, วันที่) |
| `lab_order` | ผลแลป (รหัสแลป, ผล) |
| `lab_items` | รายการแลป (รหัส, ชื่อ) |

หากต้องการรายละเอียดเพิ่มเติม ดูที่ [ภาคผนวก - โครงสร้างฐานข้อมูล](./8-appendix.md)

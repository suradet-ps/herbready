# 7. การแก้ปัญหาและคำถามที่พบบ่อย

ส่วนนี้รวบรวมปัญหาที่พบบ่อยและวิธีแก้ไข รวมถึงคำถามที่พบบ่อยเกี่ยวกับการใช้งาน HerbReady

---

## ปัญหาการเชื่อมต่อฐานข้อมูล

### ไม่สามารถเชื่อมต่อฐานข้อมูลได้

**อาการ:** แสดงสถานะ "ไม่เชื่อมต่อ" หรือแจ้งเตือน "Connection Failed"

**วิธีแก้ไข:**

1. **ตรวจสอบ MySQL Server**
   - Windows: เปิด Services ตรวจสอบว่า MySQL ทำงานอยู่
   - macOS/Linux: รัน `sudo systemctl status mysql`

2. **ตรวจสอบ Host และ Port**
   - ตรวจสอบว่า Host ถูกต้อง (localhost หรือ IP ของ server)
   - ตรวจสอบ Port (ค่าเริ่มต้นคือ 3306)

3. **ตรวจสอบ Firewall**
   - อนุญาตการเชื่อมต่อพอร์ต 3306
   - ตรวจสอบ Windows Firewall, macOS Firewall หรือ iptables

4. **ตรวจสอบสิทธิ์ผู้ใช้**
   ```sql
   -- ใน MySQL Client
   CREATE USER 'username'@'localhost' IDENTIFIED BY 'password';
   GRANT ALL PRIVILEGES ON database_name.* TO 'username'@'localhost';
   FLUSH PRIVILEGES;
   ```

### ทดสอบเชื่อมต่อสำเร็จแต่โหลดข้อมูลไม่ได้

**อาการ:** Test Connection ผ่าน แต่เมื่อโหลดข้อมูลจริงไม่ได้

**วิธีแก้ไข:**

1. ตรวจสอบว่าผู้ใช้มีสิทธิ์ SELECT บนตารางที่ต้องใช้
2. ตรวจสอบว่าฐานข้อมูล HIS มีตารางที่โปรแกรมต้องการ
3. ดูรายละเอียดข้อผิดพลาดใน Console (F12)

## ปัญหาการโหลดข้อมูล

### โหลดคิวนานมากหรือค้าง

**สาเหตุที่เป็นไปได้:**

1. ข้อมูลในฐานข้อมูลมีมากเกินไป
2. Query ไม่มี index สนับสนุน
3. เครื่อง Server ฐานข้อมูลมีปัญหา

**วิธีแก้ไข:**

1. จำกัดช่วงวันที่ให้แคบลง
2. เลือกเฉพาะห้องตรวจที่ต้องการ
3. ตรวจสอบประสิทธิภาพฐานข้อมูล (performance)
4. ติดต่อผู้ดูแลระบบฐานข้อมูล

### ไม่พบข้อมูลผู้ป่วย

**วิธีแก้ไข:**

1. ตรวจสอบวันที่ถูกต้อง
2. ตรวจสอบรหัสห้องตรวจถูกต้อง
3. ตรวจสอบว่ามีผู้ป่วยมาพบจริงในวันนั้น

---

## ปัญหาการแสดงผล

### ตัวอักษรภาษาไทยไม่แสดง

**สาเหตุ:** Font ภาษาไทยไม่ติดตั้งในระบบ

**วิธีแก้ไข:**

1. ติดตั้ง Font ภาษาไทย (เช่น Sarabun, TH Sarabun)
2. รีสตาร์ทโปรแกรมหลังติดตั้ง Font

### หน้าจอไม่พอดีกับขนาดจอ

**วิธีแก้ไข:**

1. ปรับขนาดหน้าต่างโปรแกรม
2. ใช้คีย์บอร์ด Ctrl +/- เพื่อซูมเข้า-ออก

---

## ปัญหาการพิมพ์

### ไม่สามารถพิมพ์ได้

**วิธีแก้ไข:**

1. ตรวจสอบเครื่องพิมพ์เชื่อมต่อและเปิดอยู่
2. ตรวจสอบ Printer Queue ว่ามีงานค้างอยู่หรือไม่
3. ลองเลือกเครื่องพิมพ์เครื่องอื่น
4. ลองบันทึกเป็น PDF แทน

### ใบสั่งยาข้อมูลไม่ครบ

**วิธีแก้ไข:**

1. ตรวจสอบข้อมูลในฐานข้อมูลครบถ้วนหรือไม่
2. ตรวจสอบการตั้งค่ายาในโปรแกรม
3. ตรวจสอบว่าผู้ป่วยมี HN และชื่อถูกต้อง

## คำถามที่พบบ่อย (FAQ)

### Q: HerbReady ใช้งานได้กับฐานข้อมูล HIS ใดบ้าง?

**A:** HerbReady ใช้ได้กับฐานข้อมูล MySQL ที่มีโครงสร้างตารางตามที่กำหนด เช่น HosXP, PMQA หรือระบบอื่นที่ใช้ MySQL

### Q: สามารถใช้งาน HerbReady พร้อมกันหลายเครื่องได้หรือไม่?

**A:** ได้ โดยทุกเครื่องต้องเชื่อมต่อกับฐานข้อมูล HIS เดียวกัน

### Q: ข้อมูลจะถูกบันทึกที่ไหน?

**A:** ข้อมูลการตั้งค่าจะถูกบันทึกในเครื่องของผู้ใช้ (Local) ส่วนข้อมูลผู้ป่วยอ่านจากฐานข้อมูล HIS โดยตรง

### Q: สามารถสำรองข้อมูลการตั้งค่าได้หรือไม่?

**A:** ได้ โดยใช้ฟีเจอร์ "ส่งออก" ในการตั้งค่า เพื่อสำรองไฟล์ app_config.json

### Q: หากเปลี่ยนเครื่องคอมพิวเตอร์ ต้องตั้งค่าใหม่หรือไม่?

**A:** ใช่ ต้องตั้งค่าการเชื่อมต่อฐานข้อมูลและการตั้งค่ายาใหม่ หรือนำเข้าไฟล์ app_config.json ที่สำรองไว้

### Q: รองรับการใช้งานแบบออฟไลน์หรือไม่?

**A:** โปรแกรมต้องเชื่อมต่อฐานข้อมูล HIS จึงใช้งานได้ ไม่รองรับออฟไลน์

### Q: มีกี่ภาษาให้เลือก?

**A:** รองรับ 2 ภาษา: ไทย และ English (อังกฤษ)
